# 機能

Hover Translate は、読む流れを止めないことを前提に設計されています。ブロックにホバーし、翻訳を読み、そのまま読み進められるように、UI は最小限に保たれています。

## 言語の自動判定

Hover Translate は、ホバーしたテキスト自体を見て翻訳方向を決定します。

- テキストに日本語文字が含まれていれば、日本語から英語への翻訳として送信します。
- 日本語文字が含まれていなければ、英語から日本語への翻訳として扱います。
- ソース言語を手動で切り替える必要はありません。

```ts
/[\u3040-\u30ff\u4e00-\u9fff]/
```

この正規表現で、ひらがな・カタカナ・漢字を含むかどうかを確認してから DeepL に送信します。

## ブロック単位の検出

単語の一部ではなく、最寄りのブロック要素全体を翻訳対象にします。

- 対応する要素には `p`、`li`、`h1` から `h6`、`td`、`blockquote`、`article`、`section`、`div`、`main`、`aside`、`figcaption`、`dd`、`dt` などがあります。
- 単語単位よりも、文や段落単位のほうが文脈を保った翻訳になりやすくなります。
- 300 ms のホバーディレイにより、カーソルが通過しただけの意図しない発火を抑えます。

## ページ内ツールチップ

翻訳結果は、ホバーした要素の近くにフローティングツールチップとして表示されます。

- ツールチップは `position: fixed` で表示されます。
- 画面端に近い場合は、見切れないように上下左右の位置を自動で調整します。
- ホバーが外れるとツールチップは消えます。
- エラー時は左側の赤いボーダーとエラーメッセージで通常表示と区別されます。

## LRU 翻訳キャッシュ

同じテキストを同一セッション中に何度も再翻訳しないようにしています。

- 背景側の Service Worker に、100 件まで保持する LRU キャッシュがあります。
- 同じテキストへの繰り返しホバーでは、キャッシュから即座に結果を返せる場合があります。
- これによりリクエスト数を減らし、DeepL Free の月間クォータを節約できます。
- キャッシュは best-effort の仕組みです。Service Worker が再起動すると、メモリ上のキャッシュは消去されます。

## キーボードショートカット

既定のショートカットは `Alt+Shift+T` です。

- 通常のページ上で、Hover Translate を ON/OFF できます。
- 切り替え直後には、アクティブタブの右上に `Hover Translate: ON` または `Hover Translate: OFF` というトーストが表示されます。
- 別のキーにしたい場合は、ポップアップの **Change** リンクから `chrome://extensions/shortcuts` を開いて変更できます。

## クォータ表示

ポップアップには、DeepL の利用状況を確認するためのクォータ表示があります。

- ポップアップを開くたびに、DeepL の `/v2/usage` エンドポイントから最新の利用量を取得します。
- 結果はプログレスバーで表示されます。
- 80% を超えるとバーはアンバー系の警告色に変わります。
- 95% を超えると赤系の危険色に変わります。
- **Refresh** を使うと、ポップアップを閉じずに再取得できます。

## 最大文字数のガードレール

大きすぎるブロックは、API に送る前にクライアント側で止められます。

- 既定値は 1500 文字です。
- **Settings** から 500 から 5000 の範囲で変更できます。
- ホバーしたブロックが上限を超える場合、コンテンツスクリプトはリクエストを送信せず、代わりに `Text too long` を表示します。
- 長い段落や大きな表が多いページでは、上限を下げるとクォータ節約に役立ちます。
